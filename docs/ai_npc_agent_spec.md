# Specification: Persistent LLM NPCs

## 1. Agent Life-cycle
NPCs are procedurally generated by the `Sector` but gain "Sentience" (LLM instantiation) when they enter a 5-light-year radius of the player. 
- **Persistence:** Their "History" and "Current Goals" are stored in the external database, keyed by a unique `AgentId`.

## 2. Memory & History
Using the ADK, each NPC maintains a **Vector Memory**:
- **Short-term:** Recent `STCOrders` received and local traffic density.
- **Long-term:** Previous interactions with the player (e.g., "Player Vanguard-01 saved me from a collision in Sol").

## 3. Tool-Calling (STC Integration)
NPCs do not follow hardcoded paths. They use LLM tool-calling to interact with GalaxyLib:
- `request_flight_plan(destination_id)`: Interacts with the `STC` to receive a `FlightSegment` list.
- `broadcast_hail(string message)`: Sends a natural language message to nearby ships.
- `evaluate_collision_risk()`: Analyzes `STC` predictive data and decides whether to obey the STC break-turn or "panic."

## 4. Personality Archetypes
- **The Freight Hauler:** Obsessed with efficiency, annoyed by STC delays.
- **The System Scout:** Looking for unique `SystemBody` features; shares data for credits.
- **The Deserter:** Ignores `ROUTE DEVIATION` orders; communicates in fearful or defiant LLM tones.
